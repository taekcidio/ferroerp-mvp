# FerroERP — Definition of Ready (DoR) (Sprint 0)

Una historia está **Ready** (lista para desarrollo) si cumple:

- [ ] Está escrita en formato **Como / Quiero / Para**
- [ ] Tiene **mínimo 5 criterios de aceptación**
- [ ] Tiene **prioridad** definida (High / Medium / Low)
- [ ] Tiene **responsable (Assignee)** asignado
- [ ] Tiene **estimación** (Story Points Fibonacci o label SP-1/2/3/5/8)
- [ ] Tiene checklist de trabajo (UI, Backend, DB, Validaciones, Testing, Docs)
- [ ] Tiene **datos de prueba definidos** (producto demo, cliente demo)
- [ ] Dependencias identificadas (si aplica)
