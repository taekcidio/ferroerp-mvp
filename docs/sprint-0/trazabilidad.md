# FerroERP — Matriz de Trazabilidad (Sprint 0)

## Objetivo
Garantizar trazabilidad entre:
Historia → Criterios de aceptación → Casos de prueba → Evidencia (commit/captura)

| Historia | AC (1..5) | TC (1..5) | Evidencia (commit/captura/link) |
|---|---|---|---|
| Registrar productos | AC1-AC5 | TC1-TC5 | |
| Entrada de stock | AC1-AC5 | TC1-TC5 | |
| Salida de stock | AC1-AC5 | TC1-TC5 | |
| Consultar stock | AC1-AC5 | TC1-TC5 | |
| Registrar cliente | AC1-AC5 | TC1-TC5 | |
| Consultar clientes | AC1-AC5 | TC1-TC5 | |
| Registrar interacción (opcional) | AC1-AC5 | TC1-TC5 | |

## Cómo llenar “Evidencia”
Ejemplos válidos:
- Link a commit: `https://github.com/.../commit/<hash>`
- Link a PR: `https://github.com/.../pull/<id>`
- Captura subida a `docs/evidencias/` y referenciada aquí
