# FerroERP — Cronograma (Sprint 1) (Sprint 0)

## Objetivo
Planificar el trabajo del Sprint 1 incluyendo desarrollo, integración y pruebas.

## Plan recomendado (1 semana)
- **Día 1 (Planning):**
  - Refinamiento final de historias (DoR)
  - Estimación (Planning Poker / Fibonacci)
  - Checklist de tareas y asignación a 4 personas
- **Día 2:**
  - Inventario: Registrar productos + Entrada de stock (inicio)
  - QA define plantilla de pruebas y empieza TC
- **Día 3:**
  - Inventario: Salida de stock + Consultar stock
  - CRM: Registrar cliente (inicio)
- **Día 4:**
  - CRM: Consultar clientes + (opcional) Registrar interacción
  - Integración y ajustes
  - Ejecución de pruebas completas
- **Día 5 (Cierre):**
  - Evidencias (capturas/links/commits)
  - Actualización de trazabilidad
  - Sprint Review + Retro

## Notas
- El trabajo debe verse reflejado en Issues/Project y commits del repositorio.
- Testing no se deja para el final: se ejecuta desde el día 2.
